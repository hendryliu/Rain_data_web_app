# Singapore Rainfall Web App — Workflow

## Prerequisites

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) (Python package manager)
- Docker & docker-compose (for deployment)

## Project Structure

```
Rain_data_web_app/
├── data/                        # Raw CSV files (2016–2024, ~7 GB)
├── scripts/
│   └── preprocess.py            # CSV → Parquet + stations.json
├── app/
│   ├── main.py                  # FastAPI server
│   ├── queries.py               # Pre-built query functions + registry
│   ├── llm.py                   # LLM integration (llama.cpp)
│   └── static/
│       └── index.html           # Leaflet map + Plotly chart + chat UI
├── processed/                   # Generated by preprocess.py (git-ignored)
│   ├── stations.json
│   └── rainfall/
│       └── {station_id}.parquet
├── models/                      # GGUF model files for llama.cpp (git-ignored)
├── requirements.txt
├── Dockerfile
├── docker-compose.yml
├── build-up.md
└── .gitignore
```

## Local Development

### 1. Set up the environment

```bash
uv venv .venv
uv pip install -r requirements.txt
```

### 2. Preprocess the data

Converts raw CSVs into per-station Parquet files and a `stations.json` index.

```bash
uv run python scripts/preprocess.py
```

This creates the `processed/` directory with:
- `stations.json` — list of 91 stations with id, name, longitude, latitude
- `rainfall/{station_id}.parquet` — one Parquet file per station with timestamp and reading_value columns

### 3. Start the dev server

```bash
uv run uvicorn app.main:app --reload
```

Open http://localhost:8000 in your browser.

### 4. Use the app

- The map shows all rainfall stations across Singapore
- Click a station marker to see its name and load its rainfall chart
- Use the year dropdown to filter by year (2016–2024) or view all years
- Use the chat sidebar on the right for data analysis (see below)

### 5. Chat features (quick queries)

The chat sidebar provides pre-built query buttons that work without any extra setup:

1. Select a station on the map
2. Optionally select a year from the dropdown
3. Click any quick-query button: Station Stats, Monthly Totals, Yearly Totals, Top Rainy Days, Dry Spell, Hourly Pattern, Rainiest Week, or Compare Stations

Results appear inline as charts, tables, or text.

### 6. Chat features (free-form natural language — optional)

To enable natural language queries, you need a local LLM via llama.cpp:

1. Download a GGUF model (recommended: Llama 3 8B Instruct, Q4_K_M quantization, ~5 GB):
   ```bash
   mkdir -p models
   # Download your chosen model into models/model.gguf
   ```

2. Start the llama.cpp server:
   ```bash
   # Option A: Run llama.cpp directly
   llama-server -m models/model.gguf --host 0.0.0.0 --port 8080 --ctx-size 4096

   # Option B: Use Docker (see Docker Deployment section)
   ```

3. The chat text input will activate automatically when the LLM is detected at startup.

You can then type questions like "What was the rainiest day at Clementi in 2023?" and the LLM will map your question to the appropriate query.

## API Endpoints

| Endpoint | Description |
|---|---|
| `GET /` | Serves the frontend (index.html) |
| `GET /api/stations` | Returns all stations as JSON |
| `GET /api/rainfall/{station_id}?year=` | Returns rainfall time series for a station, optionally filtered by year |
| `POST /api/chat` | Chat endpoint — accepts pre-built query or free-form message, returns structured results |
| `GET /api/chat/queries` | Lists available pre-built queries with parameter schemas |
| `GET /api/chat/health` | Checks if the LLM backend (llama.cpp) is reachable |

## Docker Deployment

> **Note:** Run the preprocessing step locally first — the Docker image copies from `processed/`.

### Web app only (quick queries work, no LLM)

```bash
docker compose up --build
```

### Web app + LLM (full features)

```bash
docker compose --profile llm up --build
```

This starts both the web app and a llama.cpp server. Make sure you have a GGUF model file at `models/model.gguf` before starting.

The app will be available at http://localhost:8000.
